#@version: 2025-10-18a

あなたは製造保証PFMEAの整合性チェックを行う工程設計担当です。以下の入力を使って、工程の機能と製造保証項目を順序どおりに追跡し、既存PFMEAの要求事項へマッピングしてください。

${function_list}

${assurance_list}

要求事項は全部で ${requirement_count} 件あります。インデックス 1 から ${requirement_count} まで抜け漏れなく割り当ててください。
${requirement_list}

追加メモ:${extra_notes}

## 用語とデータ粒度
- **工程の機能**: 工程で実現する目的／メカニズム。例（実サンプルSheet1より）「ｹｰｽを加工位置まで搬送し、ﾊﾞｯｸｱｯﾌﾟでｹｰｽを保持」など。通常は数件。
- **製造保証項目（assurance）**: その工程で守るべき品質ガードレール。サンプルでは 3〜5 件程度で、「ｽﾗｲﾄﾞﾄﾞｱｷﾞﾔｽﾞﾚなきこと」「ﾜﾚ,ｶｹ,白化なきこと」のように抽象的に定義されます。`assurance_index` はこの一覧の行番号（${assurance_index_range}）を指します。保証項目が存在しない工程のみ 0 を使用できます。
- **要求事項（PFMEA行）**: 故障モード単位の詳細行で、同じ文章に `#1`, `#2` のような連番が付くことがあります。要求事項は数十件になるため、同じ `assurance_index` を複数回使うのが正常です。
- 同じ文章でも `#番号` が異なれば別の要求事項として扱い、重複割り当ては禁止です。

## マッピングガイド
1. 工程の機能を順番に読み、対応する製造保証項目を確認してください。保証項目の行番号を `assurance_index` に入力します。
2. 要求事項を上から順に評価し、「どの機能／保証の組み合わせで守るべきか」を判断して `function_index` / `assurance_index` を設定します。1つの保証項目に複数の要求事項を割り当てても構いません。
3. どの保証項目にも該当しない要求事項がある場合のみ `assurance_index=0` を使用し、`reason` に不足している保証の概要を書いてください。
4. 返却前に自分で検証し、`function_index`・`assurance_index`・`requirement_index` がそれぞれ 1〜件数の範囲内かチェックします。範囲外や `requirement_index` の飛び番がある場合は修正後の JSON だけを返してください。

### 参考例
例: ｽﾗｲﾄﾞﾄﾞｱ組付（実サンプルSheet1）

| requirement_index | requirement | function_index | assurance_index | 備考 |
| --- | --- | --- | --- | --- |
| 1 | ｹｰｽ搬送ｱﾀｯﾁﾒﾝﾄ把持機構が正常に機能し、ｹｰｽがｶﾞﾀ無く把持される事 #1 | 1 | 1 | 要求は「把持がズレないこと」なので保証#1に紐づけ |
| 6 | ｹｰｽがﾊﾞｯｸｱｯﾌﾟにより確実に保持される #1 | 1 | 2 | バックアップ保持は保証#2に割り当て |
| 10 | A/Mｼｬﾌﾄ位置調整機能... #1 | 2 | 3 | ｷﾞﾔ噛合い品質は保証#3 |

このように、要求事項が 57 件あっても `assurance_index` は 1〜3 の間で再利用されます。7 や 9 のような存在しない番号は使いません。

## 方針
- 工程の機能と製造保証項目は、上から順番に進み前の項目へ戻らない。
- すべての要求事項を1回ずつ割り当て、同じ要求事項を複数の機能へ割り当てない。
- 各要求事項を最も関連性の高い機能／製造保証項目に紐づける。
- 必要に応じて同じ機能に対して複数の要求事項を割り当ててよい。
- 原文の表現を維持したまま、理由には判断根拠を簡潔にまとめる。

## 出力形式
次のJSONだけを返してください。追加の説明文は不要です。要求事項の件数と同じだけ `records` 要素を必ず返してください。
```json
{
  "records": [
    {
      "function_index": 1,
      "assurance_index": 1,
      "requirement_index": 1,
      "function": "工程の機能をそのまま記載",
      "assurance": "製造保証項目をそのまま記載（該当なしの場合は空文字）",
      "requirement": "要求事項をそのまま記載",
      "reason": "対応付けの根拠を30〜120文字で記載"
    }
  ]
}
```

### 制約
- インデックスは1から始まる整数を使用する。
- `assurance_index` は製造保証項目が無い場合のみ 0 を使用し、それ以外は ${assurance_index_range} の範囲に収める。存在しない番号・`requirement_index` の複写・空文字は禁止。
- 返却する前に `function_index` / `assurance_index` / `requirement_index` の範囲チェックを必ず行い、異常があれば応答を修正する。
- `reason` には工程条件や要求内容とのつながりを記述する。
- JSON以外の文字列を返さない。
